# Drive — облачное файловое хранилище с веб-интерфейсом

![License](https://img.shields.io/badge/License-GPLv3-green)

[🇬🇧 English (Translation)](./README.md) | 🇷🇺 Русский язык

Локальное веб-приложение для хранения и управления файлами с поддержкой пользователей, разграничением доступа и административной панелью.
Backend реализован на **чистом** Node.js (без фреймворков), frontend — vanilla JavaScript + SCSS.

---

## 🚀 Возможности

### 🔐 Аутентификация

* Регистрация и вход пользователей
* Хеширование паролей (PBKDF2)
* Сессионная модель доступа
* Rate limiting на auth-эндпоинты

### 📁 Файловый менеджер

* Загрузка файлов (включая drag & drop)
* Просмотр структуры папок
* Скачивание и удаление файлов
* Организация файловой иерархии

### 🛡 Система прав доступа

Битовая модель разрешений:

* **R** — чтение
* **C** — изменение
* **A** — администрирование

Гибкая комбинация прав на уровне пользователей.

### 👥 Администрирование

* Управление пользователями
* Изменение прав доступа
* Удаление аккаунтов

### 🔒 Безопасность

* HTTPS (self-signed TLS сертификат)
* CSRF-защита
* Ограничение частоты запросов
* Изоляция пользовательских данных

## ⚙️ Установка и запуск

### 📌 Требования

* Node.js ≥ 18
* OpenSSL (для генерации TLS сертификата)

---

### 1. Клонирование репозитория

```bash
git clone https://github.com/ZZakharC/drive.git
cd drive
```

---

### 2. Генерация TLS сертификата

Приложение использует HTTPS и требует `server.key` и `server.crt`:

```bash
openssl req -x509 -newkey rsa:4096 \
  -keyout server.key \
  -out server.crt \
  -days 365 \
  -nodes \
  -subj "/CN=localhost"
```

---

### 3. Запуск сервера

```bash
node server.js
```

По умолчанию приложение доступно:

```
https://localhost:3000
```

Конфигурация:

```
server/config.js
```

Там задаются:

* порт
* пути хранения
* лимиты запросов
* время жизни сессий
* включение регистрации
* и т. д.

---

### 4. Первый запуск

* Откройте веб-интерфейс в браузере
* Зарегистрируйте пользователя (если включено `registerOn`)
* По умолчанию: права `R + C`, но **первый пользователь получит все права**!

## 📂 Архитектура проекта

```
drive/
├── client/
│   ├── html/        # страницы (login, register, drive, admin)
│   ├── js/          # frontend логика (auth, file manager, admin)
│   ├── scss/        # исходные стили
│   ├── style/       # собранный CSS

│

│   ├── app.js       # HTTP(S) маршрутизация
│   ├── config.js    # конфигурация приложения

│   ├── users.js     # пользователи и сессии

│
├── storage/         # пользовательские файлы
├── users.json       # база пользователей
├── server.key       # TLS ключ
├── server.crt       # TLS сертификат
└── server.js        # entry point
```

## 🔐 Модель прав (битовая маска)

| Бит | Право      |
| --- | ---------- |
| 1   | Read (R)   |
| 2   | Change (C) |
| 4   | Admin (A)  |

Пример:

* `3` = R + C
* `7` = R + C + A
